# 音频先入工作流 (Audio-First)

这是 OpenTake 的默认剪辑标准。无论题材是口播、Vlog、混剪还是综艺包装，**先把音频铺好，再对着音频切画面**——这是让成片节奏稳、过渡顺的最可靠路径。

## 时间线是用来做什么的

时间线(Timeline)是这条片子的**时间地图**：每条轨道(Track)按先后顺序承载一串片段(Clip)，每个片段占据 `[startFrame, startFrame + durationFrames)` 这段帧区间。你的工作不是"把素材摆上去"，而是**安排它们在时间上的先后、长短和叠放关系**，让观众在正确的时刻听到该听的、看到该看的。

- **视频轨**(V1、V2…)：从下往上叠放，**编号越大越靠上、越后绘制**——V2 会盖在 V1 上面。主画面放 V1，B-roll/示意图/字幕叠在 V2 及以上。
- **音频轨**(A1、A2…)：承载口播/旁白(A1)与 BGM/音效(A2…)。音频不分上下层，按内容混合。
- 一个片段有 `trimStart/trimEnd`(取源素材的哪一段)、`speed`、`volume`、`opacity`，这些决定它"播什么、播多快、多响、多透"。

## 为什么先铺音频

口播/旁白/音乐的节奏，**决定了画面应该在哪一帧切**。如果先把画面铺满，再回头硬塞音频，节奏几乎一定会垮——你会被迫去迁就已经摆好的画面长度。反过来，先把音频剪顺，画面只需要"对齐到音频的节拍"，过渡会自然得多。

所以默认顺序永远是：

1. **先铺音频** — 先导入并放好口播/旁白(A1)与 BGM(A2)。它们是整条时间线的**时间基准**。
2. **精剪口播** — 先转写(`add_captions`)拿到时间戳，再在气口、句界、废话处 `split_clip`，用 `ripple_delete_ranges` 删掉啰嗦/重复/口误并闭合空隙。**先把声音的节奏剪顺。**
3. **对着音频铺画面** — 主画面放 V1，B-roll 叠 V2，每段画面的时长**对齐已经剪好的那段音频**。
4. **混音收尾** — 口播说话时把 BGM 压低(ducking，约 10–20%)；段落过渡优先用 J-cut/L-cut。

## 如何同时操作时间线与音频轨

把音频轨当作"节拍器"，画面轨去对齐它：

- 把口播/旁白放在 A1 并**尽量锁定**(它是时间基准，不要随手移动)。
- 在 A1 上完成精剪后，**画面段的起止帧去对齐 A1 上对应内容的起止帧**——而不是相反。
- **J-cut(声音先入)**：下一段的声音比画面**先**进来几帧，让过渡有预告感。
- **L-cut(声音延续)**：上一段的声音**延续**到下一镜的画面里，让衔接更柔。
- BGM 在 A2 全程铺底；遇到人声就用 `set_clip_properties` 把 BGM 的 `volume` 压低，人声结束再抬回来。

## 多轨叠放顺序

- **画面从下往上叠**：V1 主画面 → V2 B-roll/示意图 → 更高层放字幕/标题/角标。上层遮下层，所以需要"露出底层"时用透明度或蒙版，而不是把它放到下层。
- **音频按角色分轨**：A1 人声、A2 音乐、可再加一轨音效；混音时各轨独立控制音量。
- 字幕/标题这类文字叠层放在最上层视频轨，保证不被画面盖住。

## 速查规则

**该做**

- 永远先铺音频，再对着音频切画面。
- 口播/旁白放音频轨并锁定，作为时间基准。
- 过渡优先 J-cut / L-cut。
- 人声出现时压低 BGM(ducking)。

**别做**

- 别先铺满画面再回头硬塞音频。
- 别让 BGM 盖过人声。
- 别在没有音频基准的情况下凭感觉定画面长度。
